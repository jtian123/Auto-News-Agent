# Project: Multi-Campus Culture Intelligence Agent
Prototype focus: USC / Los Angeles
Architecture: Parallelized subagents + Aggregator
Primary data source: Google Gemini Web Search tools

============================================================
0) CORE IDEA
============================================================

This system is a GENERAL culture-intelligence engine that operates
based on a CAMPUS PROFILE.

- Core engine = stable, reusable
- Campus profile = config that defines:
  • school identity
  • geography
  • trusted sources
  • query templates
  • scoring weights
  • output tone & visual style

USC/LA is the first campus profile.

============================================================
1) HIGH-LEVEL FLOW
============================================================

Weekly run (per campus profile):

1) Load campus profile (usc_la.json)
2) Spawn subagents in parallel using profile-defined queries
3) Each subagent returns structured candidate items
4) Aggregator:
   - dedupes
   - scores
   - enforces category diversity
5) Select final 6–10 items
6) Generate:
   - poster text (space-constrained)
   - Nano Banana poster prompts
   - Instagram captions
7) Export artifacts

============================================================
4) SUBAGENTS (PARALLELIZED)
============================================================

Subagents are GENERAL.
Their behavior is driven by campus profile + query templates.

Each subagent:
- runs 3–6 Gemini web searches
- extracts CandidateItems
- returns JSON list

------------------------------------------------------------
4.1 USC_OFFICIAL_EVENTS
------------------------------------------------------------
Query templates:
- "site:usc.edu events this week"
- "{SCHOOL_ALIAS} student events calendar this week"
- "{SCHOOL_ALIAS} workshop networking free this week"

Extraction rules:
- Must include date + location
- Prefer official USC pages
- Drop vague announcements

------------------------------------------------------------
4.2 USC_SPORTS
------------------------------------------------------------
Query templates:
- "{SCHOOL_ALIAS} Trojans game this week"
- "{SCHOOL_ALIAS} athletics schedule upcoming"

Extraction rules:
- Only 1–2 biggest games
- Emphasize home games or major rivals

------------------------------------------------------------
4.3 LA_EVENTS_POPUPS
------------------------------------------------------------
Query templates:
- "{CITY} pop-up event this weekend"
- "{CITY} festival this week"
- "{CITY} concert this week under $30"

Extraction rules:
- Prefer transit-accessible events
- Avoid generic “top 20” lists

------------------------------------------------------------
4.4 FOOD_DRINK_NEAR_CAMPUS
------------------------------------------------------------
Query templates:
- "new restaurant near {SCHOOL_ALIAS}"
- "{CITY} food pop-up this weekend"
- "student food deals {CITY}"

Extraction rules:
- Max 2 candidates
- Must feel special / time-limited

------------------------------------------------------------
4.5 TRANSIT_SAFETY
------------------------------------------------------------
Query templates:
- "{CITY} Metro service advisory this week"
- "street closure near {SCHOOL_ALIAS}"

Extraction rules:
- Only official sources
- Drop minor disruptions

------------------------------------------------------------
4.6 CULTURE_TREND_SCAN
------------------------------------------------------------
Query templates:
- "{SCHOOL_ALIAS} viral event this week"
- "{SCHOOL_ALIAS} campus trend"

Extraction rules:
- Must be corroborated
- Skip rumors / unverifiable memes

============================================================
5) AGGREGATOR LOGIC
============================================================

5.1 Deduplication
Duplicate if:
- same event name + date + venue
- or share canonical official link

Keep version with:
- higher confidence
- clearer date/time
- better posterability

------------------------------------------------------------
5.2 Scoring (0–100)
------------------------------------------------------------

freshness_score = max(0, 1 - freshness_days / 8)

credibility_score:
- +0.4 official domain
- +0.2 reputable venue/news
- +0.2 multiple sources
- capped at 1.0

posterability_score = posterability / 5

score =
  100 * (
    w.relevance * relevance +
    w.excitement * excitement +
    w.freshness * freshness_score +
    w.credibility * credibility_score +
    w.posterability * posterability_score
  )

------------------------------------------------------------
5.3 Selection
------------------------------------------------------------
- Sort by score desc
- Enforce weekly_constraints from campus profile
- Select final_picks

============================================================
6) POSTER + PROMPT GENERATION
============================================================

6.1 Poster Text Rules
- Minimal text
- Date + place always visible
- Designed for small-screen glance

- {details_line1}

Visual motif: {visual_motif}

"""

6.3 Instagram Caption Template
------------------------------------------------------------

- soft CTA (save / send / check out)

  "Weekly USC x LA picks by Apateu"

============================================================
8) SAFETY & QUALITY GUARDRAILS
============================================================

Drop item if:
- missing date or location
- confidence < 0.5
- only one low-quality source
- sensitive or unverifiable

============================================================
9) WHY THIS SCALES
============================================================

To add a new campus:
- create new campus_profile.json
- reuse all logic
- no prompt rewrites required

USC/LA is just the first instance.
